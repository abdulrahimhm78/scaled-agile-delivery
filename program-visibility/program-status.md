# Program Status (Executive Visibility)

## Purpose
Provide a concise, consistent, and objective view of program health to enable timely decision-making, risk mitigation, and alignment across leadership, product, and delivery.

## Reporting Cadence
- **Frequency:** Weekly (with ad-hoc updates for critical risks)
- **Audience:** Executives, Program Leadership, RTEs, Product Leadership
- **Source of Truth:** Jira / Azure DevOps, Program Boards, PI Objectives

## Delivery Metrics (Trend-Based)
- **PI Progress:** % Objectives achieved
- **Predictability:** Planned vs Completed
- **Throughput:** Completed items per iteration
- **Cycle Time:** Trend (improving / stable / degrading)
- **Defect Trend:** New vs resolved

> Metrics are directional and trend-focused, not performance evaluations.
